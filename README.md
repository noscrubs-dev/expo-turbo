# expo-turbo

Host-neutral XML protocol runtime foundation for Expo React Native and Rails, targeting Turbo 8.0.23 semantics.

> [!IMPORTANT]
> This repository implements an early XML tree, typed registry, React renderer with opt-in document/Frame lifecycle, and Rails XML foundation. It does not yet implement or claim complete Turbo compatibility, and no package has been published.

## Repository layout

- `src/` — public TypeScript package source.
- `rails/` — public `expo_turbo-rails` gem and non-route-owning Rails Engine.
- `example/expo/` — standalone native consumer and compatibility gallery.
- `example/rails/` — standalone Rails and Action Cable host.
- `protocol/` — shared protocol manifest and conformance fixtures.
- `docs/` — public architecture, compatibility, and release guidance.

The repository root is the publishable TypeScript package. It is intentionally not a package-manager workspace; both example applications keep independent dependency state.

## Development

The TypeScript package requires Bun 1.3.14 or newer:

```sh
bun install
bun run check
```

The Ruby scaffold requires Ruby 3.2 or newer:

```sh
cd rails
bundle install
bundle exec ruby "$(bundle show rake)/exe/rake"
bundle exec ruby "$(bundle show rake)/exe/rake" build
```

Install each example independently, then run both checks from the repository root:

```sh
cd example/expo && bun install --frozen-lockfile
cd ../rails && bundle install
cd ../..
bun run examples:check
```

See [CONTRIBUTING.md](./CONTRIBUTING.md) before proposing changes.

### Detailed implemented surface

- [x] Public-source ESM package with explicit `core`, `adapters`, `react`, `registry`, and `testing` exports.
- [x] Host-neutral fetch, form-confirmation, form-announcement, document-visit-announcement, navigation, Cable subscription-interface, strict Action Cable v1 JSON codec, pure `resolveActionCableEndpoint` admission, injected single-lifetime `ActionCableV1WebSocketAdapter`, lifecycle, visibility, focus, bounded same-document `DocumentAnchorScrollAdapter` requests, one-shot `DocumentRefreshScrollAdapter` reset, bounded root-history `DocumentHistoryScrollAdapter` restoration, style, storage, observability, clock, request-ID, and restoration-ID adapter contracts. `resolveActionCableEndpoint` maps one credential-free HTTP(S) origin plus an explicit same-origin path to a `ws:`/`wss:` URL and rejects userinfo, queries, fragments, origins with paths, and authority-changing or normalizing mount paths; it does not select hosts, carry tickets, or create sockets. The Cable adapter uses that host-supplied `ws:`/`wss:` endpoint and exact `actioncable-v1-json` protocol, sends subscriptions only after `welcome`, reports `connected` only after an exact confirmation, shares one server subscription per opaque identifier, retries active unconfirmed groups at Action Cable's 500 ms confirmation-guarantee cadence, and closes when its final local subscription leaves. A valid server `disconnect` with `reconnect: true` rotates live identifiers only after the replacement `welcome` and re-confirmation. Its optional `heartbeat` monitor uses the injected scheduler plus a host time source to observe every valid frame, replace one socket after more than six silent seconds, and avoid retrying a replacement that never receives a valid frame. An optional registry `reconnectRefresh` requester receives one frozen `{ baseUrl, scroll: "preserve", sourceKeys }` request after the first active `connected(true)` in a reconnect cycle; `sourceKeys` are logical source-node keys, never Cable credentials or messages. Initial connections, terminal transitions, and ordinary Stream `refresh` do not invoke it. Socket close/error, malformed input, an unsupported negotiated protocol, `reconnect: false`, or command failure is terminal and reported as a redacted `SubscriptionError`; the adapter owns no credentials, generic connection retry/backoff timing, authentication, URL derivation, or native lifecycle/network binding.

It captures exact source identities, waits for captured sources to leave `reconnecting`, lets an active document-level source win, otherwise reloads only eligible outermost mounted Frames in document order, and revalidates every source, Frame, and controller before each serial reload.
