# Changelog

All notable public package, gem, and protocol changes will be recorded here.

## Unreleased

- Add frozen ledger-only history visit proposals without enabling host writes or document actions. Ordinary advance proposals choose push versus same-location replacement from the canonical requested URL, explicit replacement retains the current index, redirect retargeting preserves the original method/identifier/index, and exact current-base commitment rejects forged, stale, settled, reused-identifier, invalid, and overflowing proposals without mutating current history.
- Add explicit cached-preview eligibility without rendering or consuming a visit. `DocumentSnapshotCache.getPreview()` returns fresh default-policy clones, suppresses retained `no-preview` snapshots only after refreshing their LRU recency, preserves fragment-free keys, leaves suppressed entries available through `get()` for restoration, and does not admit `no-cache` writes.
- Add an inert native `DocumentHistory` identity/restoration ledger without enabling router or visit actions. It validates and freezes managed/unmanaged host state, generates an explicit index-0 replacement entry for unmanaged state, preserves managed entries for native remount continuity, derives Turbo's forward/back traversal direction, admits both replacement identifiers sharing an index and promoted Frame entries reusing one identifier across URLs/indexes, and retains finite scroll positions per admitted identifier. Host writes, traversal subscriptions, snapshot consumption, previews, and `replace`/`restore` visits remain pending.
- Capture the exact outgoing document snapshot immediately before an admitted parsed-tree `advance` visit commit when a `DocumentVisitController` receives a cache. In-flight mutations are included; valid `2xx`/`4xx`/`5xx` XML commits capture, including commit-before-finalization errors. Refreshes, empty responses, cancellation, transport/parse failures, delegated/discarded redirects, `no-cache` documents, and still-blocked `replace`/`restore` proposals do not capture.
- Add root-owned native cache policy metadata through exact lowercase `data-turbo-cache-control="no-cache"` and `"no-preview"`. `no-cache` now skips snapshot writes without evicting an older admitted entry; `no-preview` remains cached with its policy intact for a future visit coordinator. Preview rendering and history consumption remain pending.
- Add explicit document-session snapshot interop without admitting history actions. `DocumentSession.captureSnapshot()` records the exact active URL/tree pair, cache writes now reject missing or mismatched document URLs, and `restoreSnapshot()` returns a frozen hit/miss result while reusing the established tree-replacement, disposal, revision, identity, and stale-response safety contracts. Router/history state, previews, scroll restoration, immediate active-visit cancellation, and link `replace`/`restore` remain pending.
- Add the host-neutral history foundation: production `DocumentTree.clone()` preserves exact keys/indexable structure without sharing mutable nodes or reusing mutation keys, while `DocumentSnapshotCache` retains ten independent snapshots under normalized fragment-free HTTP(S) URL keys with query identity, get/put LRU recency, clear/size inspection, and cached-copy-only `data-turbo-temporary` subtree pruning. Visit capture, previews, history restoration, scroll restoration, and `replace`/`restore` admission remain pending.
- Add Turbo 8.0.23 generated-form document links behind a host-configured `FormLinkSubmissionController`. Method/Stream marker presence now reuses the canonical form submission controller with exact link ownership, ordered query-to-hidden entries, `_charset_` rewriting, direct unsafe HTTP verbs, Stream negotiation, nonempty confirmation, and source/default/named/parent/top Frame destination resolution without a synthetic source-Frame origin. Browser-bypassed scheme/root/opt-out paths keep their existing delegation; exact link/inherited Frame actions and globally disabled native form interception fail closed before request ownership.
- Admit Turbo 8.0.23 destination-Frame action inheritance through the shared programmatic/registered-link visit path. An omitted action reads only exact lowercase `advance`, `replace`, or `restore` from the resolved destination Frame; invalid Frame values remain ordinary loads, present link non-actions explicitly mask inheritance, and promoted/external visits do not inherit. Recognized Frame-local inherited actions fail closed before request ownership until Frame history promotion exists.
- Match Turbo 8.0.23 link-action recognition without overclaiming history support. Only exact `advance`, `replace`, and `restore` are actions; every other value retains the ordinary document, Frame, or delegated path. Browser-bypassed scheme, root-delegated, and opted-out paths ignore action metadata; exact `advance` works for visitable top-level and owning-document-promoted Frame links, while visitable `replace`/`restore` and Frame-local recognized actions fail closed before request ownership.
- Match Turbo 8.0.23's browser link click predicate: absent, empty, and exact `_self` targets retain the current native document/Frame/delegation path, while every other target and any present `download` attribute fail closed before URL handling or request ownership. Disabled presence remains the earlier inert short-circuit.
- Extend explicit host-resolved `dirname` snapshots to hidden controls, preserving the direction companion immediately after the admitted hidden value or `_charset_` replacement through collection and GET/URL-encoded planning. Pin Turbo 8.0.23's manual submitter append as name/value-only without browser image coordinates or submitter direction sidecars, and record registered native controls as the string-valued equivalent of browser form-associated custom elements while File/Blob and arbitrary multi-name values remain unsupported.
- Match Turbo 8.0.23's exact Stream `method` selection. Structural `replace`/`update` actions now fail closed before mutation only for lowercase `method="morph"`, while every other value keeps plain semantics; document refresh likewise reserves only exact morph and treats absent, blank, arbitrary, or differently cased methods as ordinary replacement. Native morph and explicit refresh-scroll policies remain unsupported.
- Treat confirm-only native document-link activation like plain-link activation, matching Turbo 8.0.23: `data-turbo-confirm` alone is inert during activation across document, Frame, delegated, disabled, and request-supersession paths. This slice did not yet include method/Stream-generated form links, functional link confirmation, or confirm-based prefetch suppression.
- Add HTML-compatible select-option value fallback for explicit native snapshots. An authored string value, including `""`, wins; otherwise host-supplied option text is stripped and collapsed using only ASCII whitespace before successful-entry collection and request encoding. Hosts still own live selectedness, option order, group disabledness, and the raw text snapshot.
- Add explicit native hidden-control successful entries. Hidden values default to `""`; only ASCII-case-insensitive `_charset_` hidden names are replaced with `UTF-8`, while ordinary value descriptors remain unchanged and existing disabled/name/fieldset/datalist admission plus XML ordering still apply.
- Add strict native document-link delegation for normalized `mailto:` and `tel:` URLs through the existing host navigation adapter. Scheme links delegate before document/Frame capture from top-level, nested Frame, named-Frame, or opted-out contexts; disabled and unsupported-metadata admission still wins, fragments/raw C0 or DEL controls/credentials and every other scheme remain rejected, and no fetch ownership changes.
- Add capability-visible native datalist component roles. Explicitly registered controls under semantic datalist ancestry are omitted from successful entries regardless of external form ownership, while an explicitly selected submitter still follows Turbo 8.0.23's manual append-last behavior. React disabled/accessibility state remains unchanged because datalist ancestry bars ordinary entry construction rather than disabling the control. Automatic XML/DOM discovery, suggestion behavior, and child hiding remain unsupported and host-owned.
- Add capability-visible native fieldset/legend component roles plus a structural form-semantics adapter. Explicitly registered controls now inherit live disabled-fieldset ancestry with the first direct legend exception, independent nested scopes, external-owner physical ancestry, inherited-disabled control omission, inherited-disabled submitter rejection, and React effective-disabled accessibility updates. Automatic XML/DOM control discovery remains unsupported.
- Add Turbo 8.0.23-compatible native form interception modes: `on` by default, global `off`, and form-ancestor `optin`, with independent live `data-turbo` ancestry for internal or external submitters. Core and React expose a synchronous host interaction decision while explicit planning, proposal, submission, and retry APIs remain intentional bypasses.
- Add explicit native select snapshots: a disabled or unnamed select emits nothing, while selected options emit in host-supplied option order only when neither directly disabled nor inside a disabled `optgroup`, preserving repeated and empty values. Hosts retain selectedness/value/order/group-state ownership; XML/DOM option discovery, browser default-selection/value derivation, and the remaining successful-control matrix stay unsupported.
- Add bounded exact-ID external native form association: `form` overrides nearest ancestry, registered controls collect in whole-document order, external bindings/submitters share the declared exact form, owner removal/insertion/same-ID replacement rebinds through React, and stale submitter ownership invalidates proposal admission. Blank, missing, undeclared, and foreign owners fail closed; association does not move React/state-scope ancestry, and the remaining browser successful-control matrix stays unsupported.
- Add plain current-document Refresh Streams with target/template independence, Turbo's 150 ms trailing debounce, exact active-URL and active-visit guards, session-shared latest-20 request-ID suppression across document/Frame/recurse/canonical-form transport, and complete replacement through the existing document GET lifecycle. Morph and explicit scroll refresh policies remain unsupported.
