# Changelog

All notable public package, gem, and protocol changes will be recorded here.

## Unreleased

- Treat confirm-only native document-link activation like plain-link activation, matching Turbo 8.0.23: `data-turbo-confirm` alone is inert during activation across document, Frame, delegated, disabled, and request-supersession paths. Method/Stream-generated form links, functional link confirmation, and confirm-based prefetch suppression remain unsupported.
- Add HTML-compatible select-option value fallback for explicit native snapshots. An authored string value, including `""`, wins; otherwise host-supplied option text is stripped and collapsed using only ASCII whitespace before successful-entry collection and request encoding. Hosts still own live selectedness, option order, group disabledness, and the raw text snapshot.
- Add explicit native hidden-control successful entries. Hidden values default to `""`; only ASCII-case-insensitive `_charset_` hidden names are replaced with `UTF-8`, while ordinary value descriptors remain unchanged and existing disabled/name/fieldset/datalist admission plus XML ordering still apply.
- Add strict native document-link delegation for normalized `mailto:` and `tel:` URLs through the existing host navigation adapter. Scheme links delegate before document/Frame capture from top-level, nested Frame, named-Frame, or opted-out contexts; disabled and unsupported-metadata admission still wins, fragments/raw C0 or DEL controls/credentials and every other scheme remain rejected, and no fetch ownership changes.
- Add capability-visible native datalist component roles. Explicitly registered controls under semantic datalist ancestry are omitted from successful entries regardless of external form ownership, while an explicitly selected submitter still follows Turbo 8.0.23's manual append-last behavior. React disabled/accessibility state remains unchanged because datalist ancestry bars ordinary entry construction rather than disabling the control. Automatic XML/DOM discovery, suggestion behavior, and child hiding remain unsupported and host-owned.
- Add capability-visible native fieldset/legend component roles plus a structural form-semantics adapter. Explicitly registered controls now inherit live disabled-fieldset ancestry with the first direct legend exception, independent nested scopes, external-owner physical ancestry, inherited-disabled control omission, inherited-disabled submitter rejection, and React effective-disabled accessibility updates. Automatic XML/DOM control discovery remains unsupported.
- Add Turbo 8.0.23-compatible native form interception modes: `on` by default, global `off`, and form-ancestor `optin`, with independent live `data-turbo` ancestry for internal or external submitters. Core and React expose a synchronous host interaction decision while explicit planning, proposal, submission, and retry APIs remain intentional bypasses.
- Add explicit native select snapshots: a disabled or unnamed select emits nothing, while selected options emit in host-supplied option order only when neither directly disabled nor inside a disabled `optgroup`, preserving repeated and empty values. Hosts retain selectedness/value/order/group-state ownership; XML/DOM option discovery, browser default-selection/value derivation, and the remaining successful-control matrix stay unsupported.
- Add bounded exact-ID external native form association: `form` overrides nearest ancestry, registered controls collect in whole-document order, external bindings/submitters share the declared exact form, owner removal/insertion/same-ID replacement rebinds through React, and stale submitter ownership invalidates proposal admission. Blank, missing, undeclared, and foreign owners fail closed; association does not move React/state-scope ancestry, and the remaining browser successful-control matrix stays unsupported.
- Add plain current-document Refresh Streams with target/template independence, Turbo's 150 ms trailing debounce, exact active-URL and active-visit guards, session-shared latest-20 request-ID suppression across document/Frame/recurse/canonical-form transport, and complete replacement through the existing document GET lifecycle. Morph and explicit scroll refresh policies remain unsupported.
- Add host-admitted `ltr`/`rtl` value direction companions in deterministic control order; browser control discovery, direction inference, complete eligible-control coverage, and the remaining successful-control matrix stay unsupported.
- Add exact-live-node disabled document links with XML presence semantics, frozen ignored activation results, incumbent request isolation, runtime disable/re-enable checks, and Expo `Pressable` accessibility projection. Export `presenceCodec` so host registries admit empty and valued presence attributes without confusing them with boolean-value attributes.
- Add host-injected exact-form terminal live announcements with one at-most-once exact-node adapter attempt per revision after cleanup, redacted structured input, host-owned localized copy, Strict Mode and multiple-provider deduplication, stale/disposal isolation, and synchronous or asynchronous delivery-error reporting that does not alter form settlement. The standalone Expo example maps the contract to React Native `AccessibilityInfo` plus persistent polite/assertive web live regions; physical screen-reader evidence remains pending.
- Add exact-form terminal submission reporting and host-rendered recovery: session-shared frozen snapshots distinguish applied, empty, canceled, uncommitted failure, and committed-with-error outcomes after pending presentation clears; exact-attempt settlement blocks stale overwrites; React hosts can register a stable form boundary, dismiss results, and explicitly retry only a safe uncommitted failure with a fresh request ID and current-value replanning. `retryFailure` refuses unsafe or already-committed outcomes.
- Add host-injected async native form confirmation with submitter-first presence precedence, immutable proposal capture, strict boolean admission, cancellation-safe hidden exact-form ownership, no busy/destination/fetch state before acceptance, and stale/duplicate/supersede protection. The standalone Expo example supplies a native Alert adapter with a settling browser-confirm fallback.
- Cancel exact active form requests after the last mounted React form-scope/provider owner unmounts without disposing the host-owned collection. Ownership lives on the session-shared exact form, so Strict Mode replay and multiple providers cannot cancel an incumbent prematurely; headless callers retain explicit cancellation ownership.
- Add session-shared exact-form submission activity and React form/submitter presentation: observable busy state tied to the immutable planner-issued request, atomic duplicate prevention by default, an explicit same-form `supersede` override, exact activated-submitter pending/effective-disabled/nonempty `data-turbo-submits-with` presentation, and deterministic cleanup on success, failure, abort, or removal before response application. The standalone Expo example exercises the path; validation/focus, lifecycle events and request-mutation hooks, physical accessibility evidence, and morph-aware `422` preservation remain unsupported.
- Make `FormSubmissionController` the canonical ownership-safe transport-and-application path: retain and recheck the exact destination lease across staged application, stop undispatched actions after reentrant supersession, dispatch negotiated Streams across HTTP classifications, apply successful destination-Frame/document XML plus origin-Frame `4xx`/`5xx`, surface post-commit finalization failures, treat ordinary `204`/blank-`201` as no-ops while updating redirected empty Frame sources, enforce unsafe nonredirect document `200`, consume adapter-followed bodies once without refetching, and prevent invalid or stale responses from beginning mutation. `FormRequestExecutor` remains candidate-only, and morph/local-field preservation remains unsupported.
- Add destination-aware form transport coordination with session-scoped document/exact-Frame lanes shared across GET and form requests, pre-admission non-cancellation, exact-form destination-change supersession, reentrant/stale-owner safety, and frozen destination-bearing unapplied candidates.
- Change active-form `requestPlan` metadata to reject caller-supplied `frameId`; Frame-targeted callers must use `submissionProposal` so destination and `Turbo-Frame` metadata are derived together.
- Add atomic active-form submission proposals with exact Turbo Frame target precedence, internally derived `Turbo-Frame` metadata, browser-target rejection, React exposure, and hidden form/submitter/destination/tree-generation admission identity without fetching or changing request ownership.
- Add caller-scoped form request execution with exact-plan forwarding, lane-local cancellation/supersession, same-origin final-URL and MIME admission, single-read buffering, and classified immutable response candidates without tree mutation.
- Make the standalone Expo example install reproducibly with Bun's isolated linker, disabled peer auto-install, and a refreshed local-package dependency graph.
- Add registration-bound submitter selection and immutable active-form request planning to the core and React APIs, composing current form/submitter attributes and live XML-ordered values into a caller-owned request plan without fetching.
- Unify registry and React text handling around inherited `xml:space`, explicit default reset, CDATA preservation, XML-whitespace collapse, and deterministic diagnostic serialization.
- Add a shared programmatic Frame visit API that resolves current, named, parent, and top-level targets through the existing controller loader and navigation adapter.
- Add bounded recursive Frame extraction through same-origin intermediary sources with token matching, independent request IDs, redirected-base resolution, and loop/depth protection.
- Add injected lazy-Frame visibility observation with first-visible loading, observer cleanup, and renderer error propagation for automatic requests.
- Wire opt-in Frame controllers into the React renderer so mounted Frames connect, eager sources load, lifecycle snapshots update, and removed subtrees cancel and dispose their requests.
- Add Turbo-compatible Frame target precedence plus named, `_self`, `_parent`, and `_top` resolution, and an observable Frame controller for eager/manual loading, reload, lifecycle state, and cancellation.
- Add the same-origin Frame request loader with protocol headers, MIME validation, empty-response handling, cancellation, and request-epoch protection.
- Add exact Frame response extraction, wrapper-preserving child commits, redirected source tracking, embedded Stream processing, and missing-frame failures.
- Add ordered structural Turbo Stream dispatch with target/selector resolution, payload cloning, ID collision handling, no-op semantics, and per-action failure isolation.
- Add the static React protocol renderer, revision-cached document session, isolated node subscriptions, renderer error surfaces, and a real XML-rendered Expo gallery scenario.
- Add matching TypeScript/Ruby version baselines plus the TypeScript package subpaths, host-neutral adapter contracts, redacted protocol error taxonomy, and development inspector foundation.
