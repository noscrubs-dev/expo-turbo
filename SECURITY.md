# Security policy

## Supported versions

No version is supported yet. The repository is a scaffold and must not be used in production.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability-reporting flow for this repository. Include the affected commit/version, impact, reproduction steps, and any suggested mitigation without including real credentials or user data.

Maintainers will acknowledge a report, establish severity and scope, and coordinate a fix/advisory before public disclosure. Published versions will receive an explicit supported-version policy before `v0.1.0` is released.
